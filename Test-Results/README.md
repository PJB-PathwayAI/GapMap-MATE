# Test Results — Exercise Index

This folder contains the After Action Reviews (AARs) for every live validation exercise run against GapMap MATE. Exercises are the non-automatable test layer — they validate whether the deterministic engines *and* the companion conversation actually deliver the doctrine in practice, not just in code.

All exercises use the same persona ("Bodge" — infantry Private, 8 years' service, representative of the demographic majority per Paul's confirmation) for continuity of evidence across tests.

## Design Learning Progression

This is the cumulative behavioural doctrine that has emerged from testing, carried forward exercise to exercise:

1. **Exercise MIRROR** — Do not assume. Never infer capability or experience from branch, rank, or background.
2. **Exercise LENS** — Do not over-explain. Reflect, don't lecture; recognise the user's own language before translating it.
3. **Exercise LENS 2** — Do not interview — converse. Pacing, checkpoints, and momentum matter as much as content.
4. **Exercise PRISM** — Support the user's journey — don't impose one. Recognise whether the user needs discovery or validation, and adapt accordingly.

These are no longer isolated lessons from individual tests — they form the standing behavioural foundation for GapMap MATE. Future exercises validate *new* operational behaviours (Transition Partnership, etc.) rather than reopening established behaviour, unless contradictory evidence emerges from live users.
