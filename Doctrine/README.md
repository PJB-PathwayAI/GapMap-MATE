# Doctrine

This folder contains doctrine that emerged *from* the build — philosophy and behavioural principles generated through Operation PROOF that are distinct from the foundational artefacts (which defined what MATE must become before engineering began) and distinct from operational planning documents (which describe how phases are built).

Doctrine here is earned, not assumed. It exists because the engineering process produced it.

## How this folder differs from Artefacts/

**Artefacts/** — the 10 foundational documents written *before* engineering began. The doctrine that guided the build. Locked.

**Doctrine/** — principles that emerged *during* the build. Written because Operation PROOF produced evidence that shaped or extended the original doctrine. These documents may be in draft, under review, or locked — see individual file status.
